The `context` argument in a [procedure handler](/docs/reference/functions/create-procedure) and in middleware
is a `ServerContext`. It bundles the server-side services and adapters you reach for in a handler — the
WordPress client, logger, cookies, auth, and more.

/>

The adapter-backed fields (`getAuthUser`, `getConnInfo`, `verifyCaptcha`, `cookies`) resolve through the
adapters you register. Middleware can also add its own properties to `context` for everything downstream.
